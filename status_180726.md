# Suraksha AI ‚Äî Status 180726

## Project Overview
Full-stack crime analytics platform for Karnataka State Police Datathon 2026. Deployed on Zoho Catalyst serverless (Advanced I/O + Cron).

**Stack:** Python 3.11 (backend), Node 18 + Next.js 14 (frontend), Catalyst Data Store, Catalyst Cache, Catalyst QuickML (GLM).

---

## Architecture

```
functions/
‚îú‚îÄ‚îÄ surakSha_ai/           # Advanced I/O ‚Äî 30s chat-serving function (Python 3.11)
‚îÇ   ‚îú‚îÄ‚îÄ main.py            # Thin Controller: init SDK, RBAC, dispatch, try/except
‚îÇ   ‚îú‚îÄ‚îÄ common/            # Shared business logic (24 modules, 50+ DTOs)
‚îÇ   ‚îî‚îÄ‚îÄ requirements.txt   # zcatalyst-sdk, pydantic, sqlparse, requests
‚îú‚îÄ‚îÄ analytics_cron/        # Cron ‚Äî 15min precompute (Python 3.11)
‚îÇ   ‚îú‚îÄ‚îÄ main.py            # 4-phase pipeline: trends, hotspots, forecast, entity_resolution
‚îÇ   ‚îî‚îÄ‚îÄ requirements.txt   # prophet, scikit-learn, networkx, rapidfuzz, statsmodels
‚îú‚îÄ‚îÄ surakSha-api/          # FastAPI gateway (Python 3.12, 512MB, CORS)
‚îú‚îÄ‚îÄ chatbot/               # Node 18 Advanced I/O (legacy)
‚îú‚îÄ‚îÄ main.py                # Legacy root entry
‚îú‚îÄ‚îÄ ai/                    # Legacy AI engine
‚îú‚îÄ‚îÄ auth/                  # Legacy auth
‚îî‚îÄ‚îÄ * (other legacy stubs)
frontend/                  # Next.js 14 web client (Catalyst hosted)
```

### Data Flow
```
User ‚Üí Catalyst Web Client ‚Üí surakSha_ai (30s Advanced I/O)
                                    ‚Üï
                              Repository Interfaces (DIP)
                                    ‚Üï
                              Catalyst Data Store (26 tables)

analytics_cron (15min) ‚Üí ML compute (Prophet, DBSCAN, fuzzy ER, IPS)
                     ‚Üí writes snapshots ‚Üí TrendSnapshot, HotspotSnapshot, etc.
                     ‚Üí surakSha_ai reads precomputed via PrecomputedStore
```
